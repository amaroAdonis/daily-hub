# ADR 0001 — Registrar decisões de arquitetura

- **Status:** aceito
- **Data:** Fase 0

## Contexto

Decisões estruturais (stack, modelagem) precisam de rastro do "porquê", tanto
para o futuro quanto como evidência de raciocínio no portfólio.

## Decisão

Manter ADRs curtos em `docs/adr/`, numerados, no formato Contexto / Decisão /
Consequências. Um ADR por decisão relevante e irreversível-na-prática.

## Consequências

- Histórico claro das escolhas.
- Pequeno overhead a cada decisão grande (aceitável).
